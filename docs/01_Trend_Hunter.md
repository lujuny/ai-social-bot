# 热点捕获与选题模块

#### 1. 模块概述

**目标**：解决“写什么”的问题。全网抓取潜在的热点话题，并进行清洗去重。 **输入**：配置的RSS源、监控的社交媒体URL列表、关键词。 **输出**：结构化的“选题池”数据。

#### 2. 核心功能

- **多源采集**：支持微博热搜、知乎热榜、抖音热榜、特定公众号RSS。
- **去重与清洗**：去除广告、重复话题，提取纯文本标题和摘要。
- **热度标准化**：将不同平台的热度值（如微博的“爆”、知乎的“热度”）归一化为 0-100 的标准分。

#### 3. 技术方案

- **库**：`requests`, `BeautifulSoup4` (静态网页), `Playwright` (动态JS网页), `jieba` (中文分词).
- **策略**：
  - 使用 User-Agent 池和代理 IP 防止 IP 被封。
  - 使用 `TF-IDF` 算法提取关键词。

#### 4. 数据结构 (Table: `raw_trends`)

| **字段名**        | **类型** | **描述**                      |
| ----------------- | -------- | ----------------------------- |
| `id`              | UUID     | 主键                          |
| `source_platform` | Varchar  | 来源平台 (e.g., weibo, zhihu) |
| `title`           | Varchar  | 话题标题                      |
| `hot_score`       | Integer  | 归一化后的热度分 (0-100)      |
| `original_url`    | Varchar  | 原始链接                      |
| `fetch_time`      | Datetime | 抓取时间                      |
| `status`          | Enum     | 状态 (new, used, discarded)   |
